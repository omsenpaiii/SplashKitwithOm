Sort Algorithms

I have researched about three different sorting methods provided as following:


• Selection Sort:

It finds the minimum element in the unsorted array and swaps it with the element at the beginning which means during iteration it'll select the smallest item from the unsorted partition and move it to the sorted partition.
For Example, we have an unsorted array,
2 8 5 3 9 4 1
Here, 2 is our current minimum for the start. So, we will transverse in the unsorted array to find if there exists a number less than our current minimum we swap it. As in the abovementioned, we can see that 1 is smaller than 2 therefore it will swap their places.
By the above step, we have 1 element sorted in the LHS.
1 8 5 3 9 4 2
Now, 8 will be our current minimum, and transversing will be performed as same mentioned above and scan the remaining array for a smaller number while updating the current minimum as we progress to the end we found 2 and swap it with 8 and the rest will be sorted in the same way. performance but it is more efficient than selection and bubble sort.

• Insertion Sort:

It is not the best sorting algorithm in terms of sorts the LHS first by transferring an element at one time to construct a sorted array.
For Example, we have an unsorted array,
4 2 1 3
Step 1: We start from the second element which is 2 and check from starting whether or not it is smaller than it i.e first element which is 4.
Since 2 is smaller than 4, we will insert 2 at 4's position.
2 4 1 3
Step 2: Similarly, now we check the third element which is 1 from starting, and since 1 is smaller than we will insert 1 at 2's position and the rest will shift.
1 2 4 3
Step 3: Finally, we now pick the fourth element which is 3and check it from starting. But now 3 is greater than 1 so we will proceed to 2 and it is greater than 2 too so we proceed, at last 3 is less than 4 so we will insert 3 at 4’s position. Thus, our array is sorted in ascending order.
1 2 3 4

• Quick Sort:

It is a Divide and Conquer as well as a most efficient recursive sorting algorithm. The given array is initially divided up around the element that is initially chosen as the pivot element. This algorithm follows the following steps.
A pivot is just one of many items in an array it meets following conditions:
1. Pivot is at the correct position in the final sorted array.
2. All items to the left of the pivot are smaller.
3. All items to the right of the pivot are larger.
Step 1: Deciding the pivot point.
Step 2: Rearranging the items in the array until the pivot element is in a position where elements greater than the pivot remain on its right side and elements less than the pivot are kept on its left.
Step 3: Both the left and right subarrays are divided using the same technique. Up until there is just one element left in each subarray, this process is repeated.
Step 4: At this point, the items have already been sorted. A sorted array is then created by concatenating the elements. I have selected Quick Sort as it has an average time complexity of O(n log n) i.e. the amount of time taken by an algorithm to run, as a function of the length of the input and algorithm of complexity O(log(n)) is better so I have selected it to use within my program.
